Launchpad is a student-run software engineering club on campus that aims to simulate an industry development environment. I learned a tremendous amount at Launchpad - it was my first exposure to an AGILE based team, I attended my first scrum meeting, closed my first ticket, and learned about the various roles a software engineer could play in a project, i.e frontend, backend and devops.

What we did

As a team of 6 developers and 2 designers, we developed a mobile application called 'Ledge-it' to make daily finance management easier for university students. Our app included visualizations of monthly spending and incomes through pie charts and graphs, categorization, an easy-to-use UI, and the ability to simply scan a reciept to update spending logs. Have a look at the github repository.


What I did - Backend

We used the FastAPI framework to develop our server, and it was my first time using this framework - it turned out to be very intuitive. I setup data models and created endpoints to perform CRUD operations on our MongoDB database. I setup a GCP S3 bucket and created endpoints to upload base64 encoded images to the S3 instance as well delete existing images. This turned out to be more difficult than I initially thought and required me to read through multiple resources online before finally getting it to work.


I also worked on implementing JWT based authentication. During this process, I learned about the differences between session based and JWT based authentication, symmetric and asymmetric encryption and their pros and cons, as well the best practices for storing passwords in a database.


What I did - Frontend

This is was also my first time using React Native. I had some familiarity with React prior so React Native didn't seem very new. I worked on implementing the navigation for the app as well as components for a few screens - these included modals, pie charts and tables. I also got to work with the Reanimated library to implement a simple animation where the header shrinks as the user scrolls further down. Looking back now at the code I pushed out, I realize that there was a lot that I could abstract out and make reusable to achieve a cleaner component heirarchy.


Closing Remarks

I learned a lot, met some very smart people and got introduced to the world of software engineering. At the end of the 8 months, I had a decent idea of how software is developed in an AGILE environment, learned about several major topics in software development to further explore and gained some valuable full-stack experience.
